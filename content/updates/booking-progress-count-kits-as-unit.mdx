If you book equipment as kits, the check-in progress bar just got easier to read. Until now it always counted at the asset level, so a booking containing one 30-piece camera kit showed 30 separate items to return — and "2 / 32" looked alarming even when only one standalone item and part of one kit were still out.

You can now choose to count **each kit as a single unit** instead. In this mode every standalone asset counts as one item and every kit counts as one item, and a kit only counts as checked in once all of its assets are back. The progress bar then reads in "items" rather than "assets", with an inline tooltip explaining the difference.

The check-in itself is unchanged — you still return individual assets, and the per-asset list, scanner drawer, and CSV export always count assets individually. The setting only changes how the booking's progress bar tallies its total.

It's a workspace-wide setting that Owners and Administrators can turn on under **Settings > Bookings > Counting options** by enabling **Count each kit as a single unit**. The toggle saves automatically, and the default stays per-asset counting so nothing changes unless you opt in.
