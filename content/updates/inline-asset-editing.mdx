Editing a single asset field used to mean opening the full edit form, scrolling to the field, changing it, saving, and being redirected back to the asset page — four or more clicks for any small correction. Across an inventory cleanup or onboarding session, that adds up to hundreds of unnecessary clicks.

That's gone. On the asset detail page, every supported field shows a pencil icon when you hover over it. Click the pencil to enter edit mode, change the value, and save — the asset page updates in place and an entry is added to the activity log. **Description**, **Category**, **Location**, **Value**, and **all custom field types** (text, multi-line text, boolean, date, option, amount, number) now support inline editing. Tags remain in the full edit form for now.

Validation works exactly as it does in the full form: the **Value** input rejects non-numeric input with a clear error, locations blocked by an active kit assignment surface the conflict before saving, and required custom fields stay required. Permissions are unchanged — owners and admins can edit every field, self-service and base users see only fields with values and cannot enter edit mode.

Inline editing is currently desktop-only. On mobile, use the **Edit** button to open the full edit form — a touch-friendly inline experience is on the roadmap.
